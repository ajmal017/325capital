    # Introduction
      The Canalyst API is documented in [**OpenAPI format**](http://spec.openapis.org/oas/v3.0.3).

    # Authentication
      API tokens are used to authenticate with the Canalyst API. When making
      requests, include the token in the Authorization header using the Bearer
      scheme: `Authorization: Bearer {token}`.

    # Paging
      The default page size is 50 and the default maximum page size is 200. The
      following have custom page sizes and maximum page sizes:
      * Equity Model Historical Data Points: max page size is 500

    Page sizes can be customized per request (up to the maximum page size). Use

    the `page_size` query parameter to set the page size for a request.


    # Throttling
      The default throttling limit is 300 requests per minute. The following
      endpoints have custom throttling limits:
      * Drivers: 45 requests per minute
      * Scenario Forecast Data Points: 16 requests per minute
      * Scenario Period Data Points: 16 requests per minute
      * Scenarios: 4 requests per minute for POST requests

    Upon exceeding the throttling limit, a `429 Too Many Requests` response

    will be returned. The response includes a `Retry-After` header that

    indicates the number of seconds until requests can be submitted again.


    # Client SDK
      There are just a few simple steps to generate a client software development kit
      (SDK) for the Canalyst API that supports the language of your choice:

    1. Download and save a copy of the [OpenAPI schema
    file](/api/schema/?format=json).

    2. Navigate to the [Swagger Editor](https://editor.swagger.io/) web app.

    3. Open Swagger Editor's **File** menu then **Import file** and select the
    schema

    file you downloaded.

    4. Open Swagger Editor's **Generate Client** menu then select the language
    for which

    you'd like to download a client SDK.

    5. Unzip the downloaded client SDK file.


    # Glossary
      **Company**: traded on stock exchanges. A Company may have more than one
        CSIN associated with it, which in turn means that a Company may have
        more than one Equity Model Series associated with it.

      **Company ID**: an identifier for a Company. It's also the first 6
        characters of the CSIN.

      **CSIN**: the Canalyst Security
        Identification Number uniquely identifies Equity Model Series.

      **Data Point**: an entry for a Period in a Time Series. Analogous to a
        single data cell from a Model worksheet.

      **Driver**: a Data Point whose value directly or indirectly influences
        other Data Point values.

      **Equity Model**: a representation of a company's financials, key
        performance indicators and operating statistics. Primarily consists of
        a matrix of Data Points from all Time Series and Periods for a single
        Model Version.

      **Equity Model Series**: the collection of all Equity Models for a
        specific CSIN, ordered by Model Version.

      **Forecast**: later than, but not including, the most recent fiscal
        period. Analogous to everything to the right of the vertical black
        line in a Model worksheet.

      **Historical**: earlier than and including the most recent fiscal
        period. Analogous to everything to the left of the black line
        in a Model worksheet.

      **Historical Data Point Revision**: changes to a Data Point across Model
        Versions in an Equity Model Series, specifically an Historical Data
        Point.

      **Label**: additional information to provide more context around other
        resources. E.g. a Company may have a sector label, and a Time Series
        may have a financial label.

      **Label association**: a mapping between a Label and something else, e.g.
        a Time Series.

      **Last price**: the last price the Equity Model was traded at. Note that
        this value may be out of date with the Equity Model's Company's
        current last price. Analogous to the last price cell in the Model
        worksheet.

      **Model Version**: a string that identifies a single Equity Model in an
        Equity Model Series. Consists of the reporting period for which the
        Equity Model was prepared and a monotonically-increasing integer
        revision (e.g. \"FY2017.23\").

      **Model Version revision**: the revision number for a specific reporting
        period of an Equity Model in an Equity Model Series. Increments
        monotonically.

      **Model worksheet**: the Model worksheet from a Canalyst model in Excel.

      **Most recent fiscal period**: the latest Period in the Equity Model that
        has reported data. Analogous to the column immediately to the left of
        the black vertical line in the Equity Model's Model worksheet.

      **Period**: a reporting period for a Time Series. Analogous to a period
        column from a Model worksheet.

      **Scenario**: a what-if situation where provided Data Point changes are
        percolated throughout the Equity Model.

      **Sector**: a Label associated with a Company to provide additional
        context about which economic segment the Company belongs to.

      **Standalone Driver**: a type of Driver. Specifically, a Standalone
        Driver does not belong to a Time Series. E.g. Last Price.

      **Time Series**: an ordered collection of Data Points across a sequence
        of Periods. Analogous to a data row from a Model worksheet.

      **Time Series Category**: additional context for each associated Time
        Series. Roughly analogous to a section in a Model worksheet.

      **Time Series Driver**: a type of Driver. Specifically, a Time Series
        Driver belongs to a Time Series and is at the intersection of a Time
        Series and a Period.

      **Unit**: a measurement associated with a Time Series's values.

    description: >
      Retrieve driver information for a forecast data point


      A driver is any data point whose value either directly or indirectly

      influences the value of a target data point. Each entry has a

      `context_type` of either "standalone" or "time_series".


      This endpoint can return its results as either JSON (`application/json`)
      or in

      [Graphviz DOT format](https://www.graphviz.org/doc/info/lang.html)

      (`text/vnd.graphviz`) according to the `format` query parameter. See
      Graphviz's

      [Resources page](https://graphviz.org/resources/) for a number of viewers
      that are

      compatible with the format. Of particular note may be

      [Sketchviz](https://sketchviz.com/new), which does not require any
      download or

      installation to use.


      **Standalone drivers**


      A standalone driver is a data point that does not belong to a time series.

      For example, the company\'s most recent stock price.


      **Time series drivers**


      A time series driver is a data point that is at the intersection of a time

      series and period.

  - name: Scenarios
    description: |
      Create and retrieve scenarios for an equity model

      The list endpoint allows the creation of a _scenario_, which is used to
      simulate the effects of one or more changes to an equity model's forecast
      data points.

      **POST Request**

      Scenarios are created via POST requests containing a single scenario. A
      scenario is described as JSON data with the following structure:

          {
              "name": <unique name (string)>,
              "price": <stock price (real number as a string, optional)>,
              "changes": <changes to apply to data points (array of objects)>
          }

      **Changes**

      Each change has the following structure:

          {
              "time_series": <name or slug of the data point's time series (string)>,
              "period": <name of the data point's period (string)>,
              "value_expression": <expression describing the new value (object)>
          }

      Note that only data points from forecast periods (i.e. where the period's
      `is_forecast` field is `true`) may be changed. Also, each scenario may
      include no more than 50 changes.

      **Value Expressions**

      Value expressions allow clients to describe complex (or simple) types of
      values relating to an equity model. The different types of value
      expressions are described below.

      Note that, when a value expression is evaluated for an equity model, it
      results in a real number that is then assigned to the data point in the
      scenario.

      **Binary operator**

      A binary arithmetic operator (e.g. addition, division, exponent) to apply
      to a pair of operand sub-expressions

          {
              "type": "binary",
              "left_operand": <a value expression (object)>,
              "operator": "<a binary operator (string)>",
              "right_operand": <a value expression (object)>
          }

      Valid binary `"operator"` values are:

      * `"+"` (addition)
      * `"-"` (subtraction)
      * `"*"` (multiplication)
      * `"/"` (division)
      * `"^"` (exponent)

      **Data point**

      The value of a time series data point

          {
              "type": "data_point",
              "time_series": <a time series name or slug (string)>,
              "period": "<a forecast or historical period name (string)>"
          }

      **Literal**

      A literal value

          {
              "type": "literal",
              "value": <literal value to apply (real number as a string)>
          }

      **Price**

      The company's stock price

          {
              "type": "price"
          }

      **Unary operator**

      A unary arithmetic operator (e.g. negation, absolute value) to apply to an
      operand sub-expression

          {
              "type": "unary",
              "operator": "<a unary operator (string)>",
              "operand": <a value expression (object)>
          }

      Valid unary `"operator"` values are:

      * `"abs"` (absolute value)
      * `"-"` (negation)

      **Example POST Request**

          {
              "name": "Example Scenario",
              "price": "42.57",
              "changes": [
                  {
                      "time_series": "MO_CFSum_Dividend",
                      "period": "FY2021",
                      "value_expression": {
                          "type": "binary",
                          "left_operand": {
                              "type": "unary",
                              "operator": "-",
                              "operand": {
                                  "type": "data_point",
                                  "time_series": "MO_CFSum_DPS",
                                  "period": "FY2021"
                              }
                          },
                          "operator": "*",
                          "right_operand": {
                              "type": "data_point",
                              "time_series": "MO_RIS_ShareCount_WAB",
                              "period": "FY2021"
                          }
                      }
                  },
                  {
                      "time_series": "MO_RIS_EPS_WAD",
                      "period": "FY2021",
                      "value_expression": {
                          "type": "binary",
                          "left_operand": {
                              "type": "literal",
                              "value": "465203"
                          },
                          "operator": "*",
                          "right_operand": {
                              "type": "price"
                          }
                      }
                  }
              ]
          }
